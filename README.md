# Темзит MQTT Bridge — v0.5.0

Аддон для Home Assistant. Подключается к облачному сервису `service.temzit.ru`
и публикует все параметры теплового насоса через MQTT Discovery.

## Что нового в 0.5.0

- Добавлены параметры, отсутствовавшие в v0.4:
  - `TAmode` — режим термоаккумулятора и выбор датчика (выкл/СК/Ул/Дом)
  - `P15` — гистерезис отопления (0–30 °C)
  - `P12` — внешний нагреватель (ступени)
  - `P85` — управление циркуляционным насосом (5 режимов)
  - `P16` — гистерезис ГВС (0–30 °C)
  - `P11b0/P11b2` — дезинфекция и разморозка в БКН (checkbox)
  - `HardVersion2b1` — учитывать Тул для ГВС
  - `GVSDualMode` — выбор ККБ для ГМ2 (5 режимов)
  - `P60–P64` — полная секция солнечного коллектора
  - `P71` — наличие WiFi-термометра
  - `KKB1Type / KKB2Type` — типы компрессорных блоков (32 варианта)
  - `KKBDualLim` — порог включения ККБ2 в двухблочном режиме
  - `P91` — тип датчика протока (расширен до 7 вариантов)
  - `HardVersion2b6` — гликоль (checkbox)
  - `P87` — калибровка электросчётчика (имп/кВт)
- Расширен диапазон `P9` (Тгвс): 20–70 °C (был 30–65)
- Исправлено: `P1` теперь включает режим "Внешний" (value=5)
- Исправлено: `P88` — правильные метки процентов (55% = value 6)
- Исправлено: `P14` — погодокомпенсация теперь select 0..1.0 (шаг 0.1)

## Установка

1. Скопировать папку `temzit_mqtt` в `/addons/` на вашем HA
2. Supervisor → Add-on Store → Локальные аддоны → **Темзит MQTT Bridge**
3. Заполнить параметры:
   - `temzit_login` / `temzit_password` — от service.temzit.ru
   - `mqtt_host` — обычно `core-mosquitto`
4. Запустить аддон

## Карточка Lovelace

Содержимое `temzit_lovelace_card.yaml` вставить в HA как **Manual card**.
Имена entity формируются автоматически как `<тип>.temzit_<ключ>`.
